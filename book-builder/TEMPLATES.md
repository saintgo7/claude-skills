# 프로젝트 유형별 책 템플릿

> 사용자의 소스 문서 성격에 따라 적절한 템플릿 선택.

---

## 1. 기술 설계서 (Tech Design Book) — SAP 사용

대형 SaaS/플랫폼 프로젝트용. 10-20개 설계 문서를 17장으로 통합.

### 장 구성

```
서문 (Preface)
├── 대상 독자
├── 본서의 위상
├── 버전 이력
└── 표기 규약

Part I: Foundations (1-3장)
1. 프로젝트 개요
   - 프로젝트 요약 (이름, 목적, 납기, 예산)
   - 해결하려는 문제
   - 스코프와 단계
   - 비기능 요구사항
   - 역할 체계
   - 팀 구성
2. 사업 범위와 기능
   - 업무 프로세스
   - 기능 영역 분류
   - 핵심 기능의 밀집도
   - 지원 표준/스펙
3. 시스템 아키텍처
   - 전체 구조 다이어그램
   - 설계 철학
   - 포트/도메인 할당
   - 앱 내부 레이어
   - 데이터 흐름

Part II: Implementation (4-10장)
4. 데이터베이스 설계
5. API 설계
6. 사용자 인터페이스
7. 핵심 도메인 #1 (프로젝트별)
8. 핵심 도메인 #2 (프로젝트별)
9. 인증/권한
10. 배포/인프라

Part III: Engineering Practices (11-16장)
11. AI Agent 오케스트레이션 (또는 팀 구성)
12. 코드 아키텍처
13. 인프라 패턴 (파일/감사/이메일/모니터링)
14. 데이터 운영 (시드/테스트/변환 템플릿)
15. 개발 워크플로우 (브랜치/커밋/PR)
16. 스프린트 계획과 테스트

Part IV: Reference (17장)
17. 부록
    - 전체 문서 목록
    - 환경변수
    - 명령어
    - 약어집
    - 참고 링크
    - 재생성 가이드
```

### 각 장 평균 분량

- 서문 + 개요: 150-300 lines
- 핵심 구현 장: 200-400 lines
- 부록: 100-200 lines
- 총: 3,000-5,000 lines

## 2. API 레퍼런스 북

REST API 또는 SDK 레퍼런스 출간용.

### 장 구성

```
1. 시작하기 (Quick Start)
2. 인증 (Authentication)
3. 공통 규약
   - 응답 형식
   - 에러 코드
   - 페이지네이션
   - Rate Limit
4. 리소스별 엔드포인트
   - 4.1 Users
   - 4.2 Products
   - 4.3 Orders
   - ...
5. 웹훅 / 이벤트
6. 에러 핸들링 가이드
7. 버전 관리
8. SDK 사용
9. 마이그레이션 가이드
10. 부록 (스키마/예제/용어집)
```

## 3. 사용자 매뉴얼

최종 사용자 대상 교육/운영 매뉴얼.

### 장 구성

```
1. 시작하기 전에
2. 로그인과 계정 관리
3. 화면 구성
4. 기본 사용법
   - 4.1 검색
   - 4.2 작성
   - 4.3 조회
5. 고급 기능
6. 관리자 기능
7. 문제 해결 (FAQ)
8. 자주 하는 실수
9. 단축키 / 팁
10. 부록 (용어집, 연락처)
```

스타일: 스크린샷 많이, 체크리스트 형식, 친근한 어투.

## 6. 공통 요소 (모든 템플릿)

### 서문 (Preface) 필수 포함

- 대상 독자 (누가 읽어야 하는가)
- 본서의 위상 (SSOT 여부, 원문과의 관계)
- 버전 이력 표
- 표기 규약 (약어, 기호)

### 부록 (Appendix) 필수 포함

- 전체 문서/참조 목록
- 환경변수 (해당 시)
- 주요 명령어 (해당 시)
- 약어집
- 재생성/유지보수 가이드

### 저작권 박스 (말미)

```html
<div style="text-align:center; margin-top: 3em; color: #666; font-size: 10pt;">

© YYYY Organization · Book Title vX.Y
Total N lines · M 주요 지표
Generated by AI Agent Team

</div>
```

## 7. 장 제목 네이밍 규약

- 숫자 + 공백 + 한글/영문 (e.g., "3. 시스템 아키텍처", "3. System Architecture")
- 한/영 번호 완전 일치 (교차 참조 안전)
- ### 하위 섹션은 "3.1", "3.2" 등 dotted notation
- 이모지 사용 금지 (xelatex 렌더링 이슈)
